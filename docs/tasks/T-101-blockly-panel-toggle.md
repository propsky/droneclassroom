## 目標

讓學生在每個關卡內，可以自己切換「手動模式」和「程式模式」。學生 4-5 年級可自決。

## 背景

- v1.3 Blockly panel 仍隱藏（沿用 baseline 行為）
- v1.3 有「手動」控制（鍵盤/觸控/搖桿）但無「程式」入口
- v1.4 第二堂課（6/29）核心：學生用 Blockly 寫程式過關
- Blockly 啟用方式採 **(C) 學生自己切**（2026-06-19 Sam 拍板）

## 描述

### 1. 預設行為
- 進入關卡時，**預設手動模式**（沿用 v1.3 邏輯，避免學生一進來就看到陌生 UI）
- Blockly panel 預設隱藏
- HUD 顯示「手動模式」徽章

### 2. HUD toggle
- HUD 角落加一個明顯的 toggle 按鈕：「🎮 手動 ↔ 💻 程式」
- 學生點按鈕切換
- 切到「程式」：顯示 Blockly panel（從右側滑入），鎖定手動控制（鍵盤/觸控/搖桿暫停）
- 切回「手動」：隱藏 Blockly panel，恢復手動控制
- 切換時 drone 位置/計時不重置（學生可中途切換複習）

### 3. Blockly panel UI
- 右側 panel，寬度約 30-40% 螢幕
- 頂部有「執行」按鈕、「清除」按鈕、「範例」按鈕
- Blockly 工作區佔 panel 主要空間
- 程式執行時，「執行」按鈕 disable，顯示進度

### 4. 程式執行時鎖定手動
- 程式執行期間，鍵盤/觸控/搖桿輸入忽略
- 程式結束（成功/失敗/中止）後恢復手動
- HUD 顯示「程式執行中」狀態

## 驗收標準

1. **預設行為**：進關卡 = 手動模式，Blockly panel 隱藏
2. **Toggle 可見**：HUD 有明顯切換鈕，4-5 年級學生看圖示就能懂
3. **切換流暢**：手動 ↔ 程式切換 < 500ms，無閃爍、無 crash
4. **狀態保留**：切換時 drone 位置/計時/分數不重置
5. **手動鎖定**：程式執行中，鍵盤/搖桿輸入不影響 drone
6. **6 個關卡都支援**：1-0 ~ 1-5 全部都有 toggle 可切
7. **無 console error**

## Sam 怎麼驗收

1. 開 `http://localhost:3000/`，進入 Level 1-0
2. 預設看到手動控制（鍵盤 W/S/A/D 可動）
3. 點 HUD 切換鈕 → Blockly panel 滑出
4. 在 Blockly 拼幾個積木（先用預設的隨便一個）
5. 點「執行」→ drone 動作
6. 動作中按鍵盤 → drone 不動
7. 切回手動 → 鍵盤 W/S 可動

## 風險 / 注意

- Blockly 載入可能慢（CDN），需要顯示「載入中」狀態
- 切換時如果 drone 正在動畫中，可能造成動作殘影 — coder 自行判斷
- 國小 4-5 年級學生，要避免 toggle 太隱晦 — 用 emoji + 大字
