## 目標

把老師後台（`teacher.html`）從「所有控制區一路往下排、要上下捲」改成**分頁（tab）切換**，老師依當下要上的**教材**點一個分頁，畫面只顯示該教材的控制 + 對應的學生狀態，不再迷路。

## 背景

- 現況 `teacher.html` 把多種教材的控制堆在同一頁直向排列：一般關卡廣播（load_level / reset / race）、大亂鬥 Arena（`arena_start` + 場地/鬼數 + 排行）、之後還要加足球（T-504）。內容越來越長，老師上課時要捲來捲去找按鈕。
- Sam 要的是**教材分頁**：一次只專注一種玩法。

## 描述

### 1. 分頁結構（tab bar）
頂部一排 tab，點擊切換下方內容區。建議分頁：
- **🎓 關卡 / Blockly**：一般關卡廣播（load_level、set_mode 手動/程式、reset_all、race_start、show_message）。
- **🏟️ 大亂鬥**：Arena 控制（mode balloon/tag、場地 grid/playground、鬼數、時長、開始）+ 大亂鬥排行。
- **⚽ 足球**：足球控制（T-504：分隊顯示、指定前鋒、開賽、比分板）。
- **👥 學生總覽**（或常駐側欄）：所有上線學生 + 進度（這塊可考慮**不放進 tab、固定常駐**，因為任何教材都想看到誰在線）。

### 2. 行為
- 預設停在「🎓 關卡 / Blockly」分頁。
- 切換 tab **只切顯示**，不重置任何連線/狀態（WebSocket、Arena/足球賽局照常在背景跑）。
- 目前作用中的教材給 tab 一個狀態標示（例：大亂鬥進行中 → 🏟️ tab 標紅點），老師切走也知道哪邊正在跑。
- 純前端（HTML/CSS/JS）切換，**不需動 server**。

### 3. 樣式
- 沿用 CREAFLY 配色；tab 大、好點（老師可能用觸控筆電 / 平板）。
- 繁中標籤 + emoji（與學生端風格一致）。

## 驗收標準

1. **分頁可切換**：點 tab → 只顯示該教材控制，其他隱藏，無上下長捲。
2. **狀態不中斷**：切 tab 時 WebSocket、Arena/足球賽局、學生列表都不重置、不掉線。
3. **進行中標示**：有教材正在跑時，對應 tab 有明顯標示（紅點/高亮）。
4. **學生總覽常駐或獨立 tab**：任何時候都能看到上線學生（設計擇一，Sam 驗收時確認順手）。
5. **三種教材控制都歸位**：關卡 / 大亂鬥 / 足球各在自己 tab，功能與分頁前完全一致（無迴歸）。
6. **無 console error**；觸控可點。

## Sam 怎麼驗收

1. 開 `http://<ip>:3000/teacher`，看到頂部 tab 列。
2. 點「🏟️ 大亂鬥」→ 只剩大亂鬥控制；點「⚽ 足球」→ 只剩足球控制；點「🎓 關卡」→ 只剩關卡廣播。
3. 在大亂鬥開一局 → 切到別的 tab → 大亂鬥仍在跑（🏟️ tab 有標示）→ 切回來狀態還在。
4. 全程看得到哪些學生在線。

## 風險 / 注意

- 重排時別動到既有 `arena_*` / `broadcast` 的訊息處理邏輯，只搬 DOM + 加 tab 切換。
- 學生列表若改成常駐側欄，注意小螢幕（平板直向）排版不要擠掉內容。
