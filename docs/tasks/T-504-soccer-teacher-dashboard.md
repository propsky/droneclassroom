## 目標

在老師後台（`teacher.html`）加上**足球賽局控制**：開始一局（`soccer_start`）、看到自動分隊結果（藍/紅各誰、誰是前鋒）、即時比分板（藍 X : Y 紅 + 剩餘時間 + 勝負）。

## 背景

- `teacher.html` 已有大亂鬥控制（`arena_start` 按鈕 + 排行接收，見 arena-field / ghost-count UI 與 `arena_*` 訊息處理）。足球比照新增一塊「⚽ 足球賽」控制區。
- T-501 伺服器已會廣播 `soccer_state` / `soccer_scores` / `soccer_end` 給老師後台（`broadcastToTeachers`）。

## 描述

### 1. 足球控制區（teacher.html 的「⚽ 足球」分頁內）
- 本控制區放在 **T-506 老師後台分頁化**的「⚽ 足球」tab 內（若 T-506 尚未完成，先以獨立區塊實作，之後搬進 tab）。
- 新增「⚽ 足球賽（3v3）」區塊：
  - 顯示目前**加入足球的學生**＋自動分隊：藍隊清單 / 紅隊清單。
  - **指定前鋒（已拍板：老師指定）**：每隊清單中可點某學生「設為前鋒」→ 送 `soccer_set_striker { studentId }`；該隊前鋒顯示彩帶圖示（🎀），且**每隊只能 1 名**（點新的會取代舊的）。
  - **開始按鈕** → 送 `{ type:'soccer_start' }`（MVP 固定 1 局 3 分；時長留一個輸入框，預設 180s，方便測試調短）。開賽時若某隊沒指定前鋒，伺服器自動補（T-501 fallback）。
  - **重設賽局 / 開新場按鈕（已拍板，課堂輪替賽要用）** → 送 `{ type:'soccer_reset' }`：比分歸零、回「待開賽」、清前鋒讓老師重新指定；另一顆「換新一輪（重新分隊）」送 `{ type:'soccer_reset', clearTeams:true }`。用於一場打完換下一組學生上場，不需重啟 server。
  - （MVP 不做手動「調整隊伍」拖人換隊 — 那是階段 2 / T-507。）

### 2. 比分板
- 收 `soccer_scores` / `soccer_state` → 顯示 **藍 X : Y 紅** + 賽局狀態（待開賽 / 倒數 / 進行中 + 剩餘時間 / 已結束）。
- 收 `soccer_end` → 顯示勝負（🔵 藍勝 / 🔴 紅勝 / 🤝 平手）。

### 3. 狀態同步
- 老師後台連上時送 `soccer_state_req` 拉一次現況（仿 `arena_state_req`）。
- 與大亂鬥控制並存、不互相干擾（兩塊獨立 UI）。

## 驗收標準

1. **開賽鈕可用**：老師按開始 → 學生端進入 3-2-1 倒數、開打。
2. **分隊顯示正確**：後台看得到藍/紅各成員。
2b. **指定前鋒**：老師點某學生「設為前鋒」→ 該生變前鋒（🎀），同隊原前鋒被取消；每隊恰 1 名；學生端該機掛上彩帶。
3. **比分即時**：學生進球 → 後台比分即時更新。
4. **剩餘時間**：進行中顯示倒數；結束顯示勝負。
4b. **重設 / 開新場**：按「重設賽局」→ 比分歸零、回待開賽、前鋒清空可重指定；按「換新一輪」→ 連分隊也重洗。下一組學生可直接接著打，無需重啟 server。
5. **狀態拉取**：老師後台重新整理 / 中途連入 → `soccer_state_req` 拉回現況。
6. **不影響大亂鬥**：兩塊控制並存無衝突。
7. **無 console error**。

## Sam 怎麼驗收

1. 開 `http://<ip>:3000/teacher`，找到「⚽ 足球賽」區。
2. 幾個學生進多人足球 → 後台看到自動分隊（藍/紅 + 前鋒👑）。
3. 按「開始」→ 學生端 3-2-1 開打。
4. 學生進球 → 後台比分跳動。
5. 3 分鐘（測試縮短）到 → 後台顯示勝負。

## 風險 / 注意

- 後台時間顯示用 `endTime`（server 時間）倒數，避免依賴 client 時鐘。
- 空場或 1 人也能按開始（測試方便）— 不要硬性擋 < 6 人。
