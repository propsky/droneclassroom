## 設計重點（摘自 drone-soccer-design.md）

- **球＝無人機本身**（含球形外框）；**無共用球**。
- **3v3**：每隊 3 台，**1 前鋒（攻擊手）+ 2 防守/助攻**。
- **得分（已拍板）**：**只有前鋒（攻擊手）** 朝正向完整穿過**對方球門**（自己視角的遠端門）＝ +1（非前鋒穿過不算、不罰）。
- **前鋒標記（已拍板）**：前鋒身上掛一條**彩帶**，全場一眼認出攻擊手。
- **前鋒指派（已拍板）**：**老師後台指定**每隊 1 名前鋒（未指定則伺服器開賽自動補第一人）。
- **得分後（已拍板）**：得分隊的**前鋒**過中線退回自家半場即可再得分（MVP 簡化為前鋒，不要求全隊退回）。
- **賽制（課堂簡版）**：**1 局 3 分鐘，進球多者勝**（正式 3 局制列階段 2）。
- **場地（已拍板）**：**6m（長軸＝兩門連線）× 3m（寬）× 3m（高）** 封閉籠子，中線分兩半場；兩端架高圓環門（中心高 ~1.5m、內徑 ~1.2m）。
- **視角**：**窄邊定點視角**（站自己窄邊往場內看，近端門＝守、遠端門＝攻），高度 1.5m；`C` 鍵可切 FPV。
- **隊色**：藍隊 / 紅隊，各 3 台；前鋒加**彩帶**標記。
- **碰撞**：**MVP 先做 A 版（玩家互不碰撞）**；B 版撞擊卡位＝階段 3。

**階段 1 總計**：約 4.5 天。完成＝6 人可連線、自動分隊、**老師指定前鋒（彩帶）**、前鋒穿門計分、前鋒過中線半場重置、1 局 3 分、時間到判勝、老師後台分頁操作。

### 階段 2 — 賽制與體驗（待 MVP 完成後細拆）

- **T-507**：3 局制 + 局數/換邊 + 得分特效 + 前鋒/隊色視覺優化 + **前鋒自選（學生「我當前鋒」、每隊限 1）** + 老師**手動調隊（換隊）**/暫停。

### 階段 3 — B 版碰撞（待階段 2 完成後細拆）

- **T-508**：玩家↔玩家碰撞推擠（卡位、撞偏前鋒），讓防守真正有用（伺服器權威或 client + 校正）。

## 已拍板的決策（2026-06-27 Sam 確認）

1. **得分資格**：✅ **只有前鋒（攻擊手）穿門得分**（非前鋒不算、不罰）。前鋒身上掛**彩帶**識別。
2. **前鋒指派**：✅ **老師後台指定**（每隊 1 名；未指定則開賽自動補第一人）。學生自選排階段 2。
3. **半場重置**：✅ **前鋒過中線即可再得分**（MVP 不要求全隊退回）。
4. **場地尺度**：✅ **對齊 6×3×3m**（不沿用單人練習較大場地）。
5. **老師後台**：✅ 改 **tab 分頁**（教材切換），見 T-506。
6. **重設賽局 / 開新場**：✅ 老師後台要有「重設賽局」+「換新一輪（重新分隊）」（`soccer_reset`），課堂輪替賽用，不需重啟 server。見 T-501 §5b、T-504。
7. **❌ 不做「速降區」**：顧問曾建議「防守手站門口讓對方前鋒變慢」來補 A 版無碰撞的防守空洞，但**真實規則沒有此機制、小朋友也難理解**，Sam 拍板**不做**。
   - **MVP（A 版）防守＝純站位**：能擋視線、佔門前位置施壓，但**無實質物理阻擋**（前鋒會穿過防守手）。
   - **真正能擋的防守＝階段 3 碰撞（T-508）**。若覺得 MVP 防守太空，再評估是否把 T-508 碰撞提前。

## 顧問討論紀錄（drone-soccer-virtual-advisor，2026-06-27）

跑了一輪深入討論，重點結論：
- **A 版無碰撞的防守空洞**是最大玩法風險 → 顧問原提「速降區」補洞，**Sam 已否決**（見上 #7）。MVP 接受防守偏弱、真防守留階段 3。
- **可考慮採納的低成本體驗建議（尚未排入 task，待 Sam 決定）**：
  - 前鋒 HUD **橙色方向箭頭**指向目標門；得分後箭頭反向提示「退半場」。
  - 進場**大字角色提示**（你是前鋒/防守、攻哪個門）→ 對應「新手 5 分鐘上手」。
  - 視覺深度：機體**正下方圓形陰影**、中線**黃色發光條**、兩半場地面染淡隊色、球門環用對方隊色。
  - **得分特效**建議不等整個階段 2，整合測試後就先加（提升課堂回饋）。
- **12 人安排**：一場 3v3 只有 6 人上場 → 採輪替賽，靠「重設 / 開新場」換組（已納入 #6）。
- 顧問丟回、目前**擱置**的問題：速降區方向判定精細度（因否決速降區，不處理）。

> 這些體驗建議若要做，再開 task 或回填 T-502/T-503/T-504；目前不擋 MVP。
