## 目標

把 T-501~T-504 串起來做一次**端到端整合測試**：模擬多 client（目標 3v3）跑完「加入 → 分隊 → 開賽 → 前鋒穿門得分 → 半場重置 → 時間到判勝」全流程，產出測試報告，作為足球 MVP（階段 1）的 ship gate。

## 背景

- 本專案測試慣例：headless Chrome（CDP）多分頁 / 多 client 腳本（見 `validate-t105.js` 自帶 server 的做法、`review-t102-103.js` CDP 模式）。足球整合測試需要**多個 WS client 同時在線**。
- 可用兩種 client 來源：
  - 真實頁面（CDP 多分頁，跑真 client 邏輯）— 最真實。
  - 純 WS 腳本 client（直接送 `soccer_join` / `soccer_pos` / `soccer_goal`）— 適合壓多人、驗伺服器權威。
- 建議：伺服器規則用純 WS 腳本驗（快、可控）；視覺/視角/分身用 1~2 個真實 CDP 分頁截圖佐證。

## 描述

### 1. 伺服器規則整合（純 WS 腳本，仿 validate-*.js）
- 自帶一個 server（自訂 PORT，如 :3150），測完關掉，不干擾課堂 server。
- 模擬 6 個 client `soccer_join` → 驗自動分隊（藍 3 紅 3）。
- 老師 `soccer_set_striker` → 驗指定前鋒（每隊恰 1、可改、取消同隊舊前鋒）；未指定的隊開賽自動補第一人。
- 老師 `soccer_start` → 驗 countdown → running、`soccer_go` 廣播。
- 前鋒送合法 `soccer_goal` → 比分 +1；非前鋒 / 對自家門 / 未 armed → 不計。
- 半場重置：得分後不過中線再穿不計；前鋒過中線（送 z 過 0 的 `soccer_pos`）後恢復。
- 縮短局時（如 10s）→ 驗 `soccer_end` + winner 依比分。
- 前鋒離線 → 驗遞補。

### 2. 視覺/視角（CDP 真實分頁，1~2 個）
- 2 個分頁各進不同隊 → 截圖：隊色正確、前鋒標記、窄邊視角看得到兩門、比分 HUD。
- 互看分身會動（其中一頁飛動，另一頁分身同步）。

### 3. 迴歸
- 跑一次大亂鬥 Arena（確認足球新增的 `soccer_*` 沒破壞 Arena）。
- 一般關卡 + 單人足球練習各載入一次（確認場景/相機互斥乾淨、無殘留）。

### 4. 測試報告
- 寫 `docs/reports/T-505-soccer-mvp.md`（或 .json）：列每個驗收 case 結果 + console error 計數 + 截圖路徑。

## 驗收標準（階段 1 MVP 全綠才算過）

1. **分隊**：6 人 → 藍 3 紅 3、各 1 前鋒；遞補正確。
2. **開賽**：老師開賽 → 全場倒數 → 開打。
3. **得分**：前鋒穿對方門 +1；非前鋒 / 反向 / 擦邊 / 偽造封包不計。
4. **半場重置**：得分後須過中線才能再得分。
5. **比分 / 結束**：比分 HUD + 後台一致；時間到依比分判勝（藍/紅/平手）。
6. **多人同步**：分身位置即時、隊色 + 前鋒標記正確、窄邊視角正確。
7. **迴歸**：Arena / 一般關卡 / 單人足球不被破壞。
8. **0 console error**（多 client 全程）。

## Sam 怎麼驗收

1. reviewer 提供測試腳本輸出 + 截圖。
2. Sam（或 reviewer）用 2~3 台裝置實際進多人足球、開賽、進球，確認體感（視角、纏鬥、進球回饋）。
3. 全綠 → 足球 MVP 可給班上試打；有缺失回 T-501~T-504 修。

## 風險 / 注意

- **湊不滿 6 人**：測試與課堂都要能 2v2 / 不對稱人數跑（MVP 不強制 6 人）。
- **CDP 多分頁資源**：6 個真實分頁較重；伺服器規則優先用純 WS 腳本驗。
- **時間到判定**：用 server `endTime`，測試縮短局時加速。
