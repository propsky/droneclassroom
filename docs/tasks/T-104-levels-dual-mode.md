## 目標

v1.3 的 6 個新手關卡（1-0 ~ 1-5）每個都要能在「手動模式」與「程式模式」下都玩。

## 背景

v1.3 6 個關卡（main.js 載入，章節 Chapter 1）：
- 1-0：起飛落地（最簡單）
- 1-1：穿過 1 個圈
- 1-2：穿過 2 個圈
- 1-3：穿過 3 個圈
- 1-4：穿過 3 個圈 + 高度變化
- 1-5：穿過 3 個圈 + 旋轉

v1.4 任務：確認 6 個關卡在「程式模式」下也能用 Blockly 過關。

## 描述

### 1. 每關「程式版」過關條件
每個關卡都要有一份**參考程式解**，讓 reviewer 可以照著跑驗證。

預估參考解：
- 1-0：`takeoff → hover 2 → land`
- 1-1：`takeoff → while distance to ring > 2: moveForward 0.5 → land`
- 1-2：`takeoff → forward 2 → rotate 90 → while distance to ring > 2: moveForward 0.5 → land`
- 1-3：`takeoff → forward 2 → rotate 90 → forward 2 → rotate -90 → forward 2 → land`
- 1-4：類似 1-3 但加 `hover 1` 在中段
- 1-5：加旋轉圈圈

### 2. 模式切換不影響關卡狀態
- 學生在關卡中切手動 ↔ 程式，圈圈位置、計時、目標不變
- 程式模式失敗可以切手動救

### 3. 過關判定
- 程式模式：Blockly 程式執行後，drone 抵達 checkpoint（穿過所有圈圈+最後降落），就算過關
- 手動模式：沿用 v1.3 邏輯
- 過關時間上報到老師後台（沿用 v1.3 WebSocket）

### 4. 計時規則
- 從關卡開始計時
- 切換模式不中斷計時
- 程式執行中如果手動干擾（不應該發生，已鎖定），不額外處罰

## 驗收標準

1. **6 個關卡都支援雙模式**：1-0 ~ 1-5 都能手動 / 程式進入
2. **每關都有可跑的參考程式解**：reviewer 用參考解可過關
3. **過關判定正確**：穿過所有圈 + 最後降落 = 過關
4. **計時正確**：模式切換不重置計時
5. **老師後台收到過關時間**：6 個關卡都上報
6. **無 console error**
7. **6 關連續跑無 memory leak / state 污染**

## Sam 怎麼驗收

1. 用 reviewer 提供的參考解，6 關各跑一次（程式模式），全過
2. 6 關各跑一次（手動模式），全過
3. 進關卡後切手動 ↔ 程式 3 次，計時不重置、drone 位置不變
4. 過關時間老師後台能看到

## 風險 / 注意

- 距離感測的「圈圈」目標實作：要先確認 drone 場景有 ring checkpoints 的物件
- 過關判定要看 drone 是不是真的「穿過」圈圈（不是只走到旁邊）
- 參考解可能太簡單繞過關卡設計 — reviewer 要確認「真的是用 Blockly 解題，不是用其他方式」
